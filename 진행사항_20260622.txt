   - [수정 내용]
     - 엑셀 업로드를 동반한 저장(신규 공통 데이터 등록) 시: data_common.json에는 오직 엑셀 원본 데이터(rows, tempHumidRows)만 정제하여 저장하고, 개별 프로젝트 전용 파일(data_[프로젝트번호].json)에는 설정값을 포함한 전체 데이터를 묶어 저장합니다.

     - 메인 화면 우측 상단의 [💾 서버 저장](레이아웃 저장) 클릭 시: 공통 파일 오염 방지를 위해 항상 현장 전용 파일에만 저장하도록 고정하였습니다.

2. 🔄 뷰 모드(viewMode) 상태 저장 및 자동 로드 기능 고도화
   - [기존 문제] 리프레시할 때 단순 파일 수정 시간으로 비교하면 현장 전용으로 조용히 쓰던 대시보드가 공통 파일의 갱신으로 인해 멋대로 종합 뷰로 바뀌어 버리는 문제.
   - [수정 내용]
     - 설정 저장 및 서버 저장 시 현재 화면의 이력 상태("현장" 또는 "종합")를 파일 내부의 'viewMode' 속성에 함께 저장합니다.
     - 화면 기동 시 현장 파일(data_[프로젝트번호].json)의 'viewMode'를 가장 먼저 읽어 마지막으로 저장해 둔 이력 모드 그대로 정밀 복원합니다.
     - viewMode가 "종합"인 경우: 현장의 고유 설정은 현장 파일에서 우선 로드하되, 실적 데이터 부분만 공통 파일(data_common.json)의 데이터로 덮어씌워 렌더링함으로써 공통 엑셀 데이터의 최신성을 유지합니다.
     - viewMode가 "현장"인 경우: 현장 전용 데이터로 100% 로드합니다.

3. 🛡️ 기존 공통 파일의 잔여 설정 데이터 차단 필터 적용
