# 2026년 7월 11일 대화 및 작업 기록 (네이버웍스 로그인 연동)

## 📌 현재 문제점 (로그인 도돌이표 현상의 원인)
현재 로그인 버튼을 눌러도 접속되지 않고 계속 로그인 화면으로 되돌아오는 이유는, `auth-config.js` 파일에 **실제 회사의 인증 정보(Client ID 및 Secret)가 입력되지 않았기 때문**입니다. (현재 `YOUR_CLIENT_ID_HERE` 등 임시 텍스트가 들어가 있습니다.)

## 🛠️ 오늘(7/11) 수정한 내용 (자동 적용 완료)
1. **`index.html` 수정:** 네이버웍스 로그인이 성공하면 수동으로 '포털 입장' 버튼을 누르지 않아도 **자동으로 `portal.html`로 넘어가도록** 코드를 개선했습니다.
2. **`portal.html` 수정:** 로그인하지 않은 외부인이 URL을 치고 바로 들어오는 것을 막기 위해, 파일 최상단에 보호 스크립트(`<script src="auth-guard.js"></script>`)를 추가했습니다.

## 🚀 월요일에 직접 해주셔야 할 작업 (아주 중요!)
Client ID와 Secret은 **직원 개개인의 아이디/비밀번호가 아닙니다.** 네이버웍스 서버가 우리 '사내 포털'을 인식하기 위한 **"포털용 고유 명찰(API Key)"**입니다. 이 명찰을 달아주어야만 네이버웍스에서 직원들에게 로그인 창을 띄워줍니다.

월요일에 업무를 시작하시면 아래 3단계를 꼭 진행해 주세요.

1. **명찰 발급받기:**
   - 관리자 권한으로 [네이버웍스 Developer Console](https://developers.worksmobile.com/)에 접속합니다.
   - **API 2.0** → **앱 추가**를 클릭하여 새로운 앱(예: 중앙알텍 사내포털)을 만듭니다.
   - 이때 **Redirect URI** 항목에 실제 NAS의 접속 주소(예: `https://우리회사NAS주소/index.html`)를 적어주고, OAuth Scope는 `user.read`를 체크합니다.
   - 생성이 완료되면 **Client ID**와 **Client Secret** 문자열이 발급됩니다. 이를 복사합니다.

2. **`auth-config.js` 파일 수정:**
   - 대시보드 폴더 안에 있는 `auth-config.js` 파일을 메모장 등으로 엽니다.
   - 복사해 둔 Client ID와 Secret 값을 빈칸 부분에 붙여넣습니다.
   - `REDIRECT_URI` 항목도 개발자 콘솔에 적었던 **실제 NAS 주소**로 정확하게 수정하고 저장합니다.

3. **NAS에 덮어쓰기 후 테스트:**
   - 수정한 `auth-config.js`를 NAS 서버에 업로드(덮어쓰기)합니다.
   - 반드시 내 컴퓨터의 `file:///C:/...` 경로가 아닌, **NAS의 실제 웹 주소(`http://...` 또는 `https://...`)로 접속**하여 로그인 테스트를 진행합니다. (로컬 폴더에서 바로 열면 보안 문제로 로그인이 작동하지 않습니다.)

직원들은 회원가입 할 필요 없이 기존에 쓰던 본인의 네이버웍스 아이디/비밀번호로 로그인하면 됩니다. 단, 로그인 창이 뜨게 하려면 **관리자(대표님)께서 포털 전용 명찰(Client ID/Secret)을 `auth-config.js`에 세팅해 주셔야만** 작동합니다! 월요일에 위 안내대로 천천히 진행해 보시고 언제든 다시 불러주세요.
