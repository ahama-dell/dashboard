# 아하마 대시보드 포털 개편 및 기능 고도화 진행 사항
**작업 일자**: 2026년 6월 20일

## 🚀 완료된 주요 작업 내용

### 1. 화면 전체 사용 및 반응형 비율 자동 맞춤 알고리즘 적용
- `max-width` 1200px 제한을 해제하여 전체 디스플레이 화면을 100% 사용하도록 개편했습니다.
- 기존의 픽셀(px) 기반 좌표 및 크기 계산 방식을 브라우저 **화면 너비 기준의 퍼센트(%) 비율 방식**으로 전면 재작성했습니다.
- 창 크기가 어떻게 변하더라도 카드와 요소들의 가로/세로 비율 및 배치 간격이 깨지지 않고 완벽하게 확대/축소되도록 구현 완료했습니다.

### 2. 드래그 앤 드롭 및 이웃 카드 자동 맞춤 (Snapping) 기능 구현
- 카드를 이동하거나 모서리를 드래그하여 크기를 조절할 때, **주변에 있는 다른 카드의 상하좌우 및 중앙선에 가까워지면 자석처럼 자동으로 딱 달라붙는 스냅(Snapping) 기능**을 추가했습니다.
- 이를 통해 수직/수평을 완벽하게 일치시켜 여러 개의 카드를 깔끔하게 정렬하기 매우 쉬워졌습니다.
- 키보드의 `Shift` 키를 누른 상태로 이동/크기 조절 시 자동 맞춤을 무시하고 미세 조정이 가능합니다.

### 3. 로고 영역 재설계 및 자유 배치 구현
- 상단의 "AHAMA MONITORING CENTER" 등 불필요한 텍스트를 제거했습니다.
- 메인 타이틀인 "통합 모니터링 시스템" 헤더 영역 안에 로고를 배치하고, **일반 대시보드 카드처럼 관리자 모드에서 언제든지 드래그하여 위치를 옮기고 크기를 조절할 수 있도록 고도화**했습니다.
- 수정된 로고의 위치 및 크기 정보도 전체 레이아웃 상태에 함께 포함되어 서버로 저장됩니다.

### 4. 프리미엄 화이트(라이트) 모드 테마 전면 전환
- 사용자의 요청에 따라 전체적인 디자인 테마를 다크 모드에서 **고급스러운 화이트 모드**로 변경했습니다.
- 배경색을 밝은 그레이와 화이트 그라데이션으로 교체하고, 카드에 은은한 반투명(Glassmorphism) 효과와 부드러운 그림자를 부여하여 가독성과 심미성을 모두 만족하도록 디자인을 개편했습니다.

### 5. 데이터 서버 저장 연동 버그 수정 (데이터 영속화)
- 포털 메인 설정값이 기기별(브라우저)로 다르게 저장되는 로컬 스토리지 문제를 해결하기 위해, `save_data.php`를 활용해 서버 스토리지(NAS)로 저장하도록 연동했습니다.
- `save_data.php` 로직에 맞춰 파일 저장명이 `data_portal_index.json`으로 생성되는 구조에 맞게 화면의 로딩(`fetch`) 경로를 완벽히 일치시켜, **변경사항을 정상적으로 불러오지 못하던 버그를 해결**했습니다.
