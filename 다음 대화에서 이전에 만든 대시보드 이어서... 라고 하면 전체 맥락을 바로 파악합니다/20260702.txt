2. 🚀 신규 작업 계획: 네이버웍스(Naver Works) 연동 포털 추가
────────────────────────────────────────────────────────────────
통합 포털(index.html) 내에 네이버웍스를 직접 연동하여 사용자가 포털 화면을 벗어나지 않고 
메일 확인 및 메신저(채팅)를 바로 사용할 수 있는 전용 판넬(카드)을 추가합니다.

■ 연동 기능 요구사항
1. 메일(Mail) 판넬:
   - 포털 메인에서 로그인된 사용자의 최신 메일 목록(보낸 사람, 제목, 수신 시간 등)을 노출.
   - 안 읽은 메일 개수를 실시간으로 배지(Badge) 형태로 포털 카드에 표시.
   - 메일 클릭 시 상세 내용을 보거나 네이버웍스 메일함으로 바로 이동하는 링크 제공.

2. 메신저(Messenger) 판넬:
   - 포털 내에서 간이 메신저 창을 제공하거나, 네이버웍스 채팅방 목록을 연계.
   - 대시보드 시스템 내의 알림(예: 온습도 이상 감지, 생산실적 저장 완료 등)을 네이버웍스 봇(Bot)을 통해 특정 대화방이나 담당자에게 실시간 메시지로 자동 발송하는 연동 기능 마련.

■ 구현 방식 및 고려사항 (Technical Approach)
1. 방법 A. Iframe 임베딩 (가장 단순함)
   - 네이버웍스의 모바일 전용 웹뷰 주소를 포털 내부의 `<iframe>` 태그를 통해 띄우는 방식.
   - 단점: 네이버웍스의 보안 정책(X-Frame-Options, 쿠키 설정 등)에 의해 임베딩이 차단될 우려가 높으므로 사전 검증 필요.

2. 방법 B. 네이버웍스 Developer Console API 연동 (추천)
   - 네이버웍스 개발자 센터에서 API 연동을 신청하고, OAuth 2.0 토큰 또는 Server 인증키(JWT)를 활용하여 데이터를 가져오는 방식.
   - 포털의 백엔드(PHP 서버 또는 Python 서버)에서 네이버웍스 API를 호출해 메일 목록 및 안 읽은 메일 수를 가져와 포털 화면에 위젯 형태로 렌더링.
   - 메신저의 경우 '네이버웍스 Message Bot API'를 등록하여 시스템 알림 메시지 발송 기능을 구현하고, 사용자 간 대화는 네이버웍스 전용 주소로 팝업 처리 또는 간이 API 채팅 모듈 구현.
