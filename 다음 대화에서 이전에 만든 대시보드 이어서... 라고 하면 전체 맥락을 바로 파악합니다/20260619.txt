3. 🏗️ 아키텍처 핵심 — 공사별 프로젝트 구분 방식
────────────────────────────────────────────────────────────────

★ production_dashboard.html 파일 하나로 여러 현장을 지원
  URL의 ?project=[프로젝트번호] 파라미터로 현장을 구분함

  예: ?project=20251113-1  → data_20251113-1.json 로드/저장
      ?project=20261115-2  → data_20261115-2.json 로드/저장

  - 관리자 모드 진입: ?project=20251113-1&admin=1
    ※ 반드시 & 로 구분 (? 로 두 번 쓰면 관리자 모드 안 됨!)
    ※ admin=1은 URLSearchParams로 정확히 파싱 (버그 수정 완료)

★ 수주번호(orderNo)와 프로젝트번호 관계
  - URL ?project= 파라미터 → 데이터 파일명 결정 (예: data_20251113-1.json)
  - 설정창 "수주번호" 필드 → 엑셀 데이터 내 수주번호로 행 필터링 (별도 개념)
  - 양평동 현장의 경우 수주번호 = 20251113-1 (동일하게 설정됨)

★ 포털(index.html)에서 신규 공사 추가 방법
  - "➕ 신규 공사 추가" 카드 클릭 → 모달에서 프로젝트명/번호 입력
  - 입력 정보는 브라우저 localStorage에 저장 (새로고침 후에도 유지)
  - 카드 우측 상단 ✕ 버튼으로 삭제 가능

5. ⚠️ 주요 특이사항 및 주의사항
────────────────────────────────────────────────────────────────

[V3 보안 우회 구조]
  저장(POST)  → 8080포트 save_data.php (표준 웹 포트 → V3 통과)
  온습도(GET) → 8090포트 Python 프록시 API

[로컬 서버 저장 불가]
  python -m http.server는 PHP를 실행하지 못함
  → 💾 서버 저장 버튼 클릭 시 "PHP 모듈 확인" 에러 후 _manual.json 다운로드됨
  → 다운로드된 파일을 NAS dashboard/ 폴더에 직접 업로드하면 반영됨

[관리자 모드 URL 규칙]
  ✅ 올바름: ?project=20251113-1&admin=1   (& 사용)
  ❌ 잘못됨: ?project=20251113-1?admin=1   (? 두 번 사용 → 관리자 모드 불가)

[urllib3 버전 충돌 (NAS)]
  NAS Python의 OpenSSL 1.0.2와 urllib3 v2.x 충돌 이슈
  → pip install "urllib3<2.0" 으로 해결 (이미 적용됨)

[NAS DSM 스케줄러 주의]
  DSM 작업 스케줄러는 완료 시 백그라운드 프로세스 강제 종료
  → start.sh를 포그라운드 블로킹 모드로 실행 (python3 -u proxy_server.py)
  → 스케줄러에서 반복 실행이 아닌 단독 실행으로 등록 필요
