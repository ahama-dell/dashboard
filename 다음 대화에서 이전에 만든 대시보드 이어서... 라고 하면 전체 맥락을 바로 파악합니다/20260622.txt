5. 🏗️ 아키텍처 핵심
────────────────────────────────────────────────────────────────

[프로젝트 구분 방식]
production_dashboard.html 파일 하나로 여러 현장 지원.
URL ?project=[번호] 파라미터로 현장 구분 → data_[번호].json 로드/저장.

예:
  ?project=20251113-1  → data_20251113-1.json
  ?project=20261201-1  → data_20261201-1.json (신규 현장 추가 시 자동 생성)

[데이터 파일 분리 구조 (2026-06-22 확정)]
① data_common.json       : 엑셀 원본 실적 데이터(rows, tempHumidRows)만 저장
② data_[프로젝트번호].json : 현장 전용 설정값 + 전체 데이터 저장

[viewMode 개념]
"현장" 모드: 현장 전용 파일로 100% 로드
"종합" 모드: 설정은 현장 파일, 실적 데이터만 data_common.json에서 덮어씌워 렌더링
→ 파일 내 viewMode 속성으로 저장/복원됨

[데이터 저장 규칙]
- 엑셀 업로드 저장: data_common.json(엑셀 데이터만) + data_[번호].json(전체) 동시 저장
- 단순 설정 수정: data_[번호].json에만 저장 (공통 파일 미접촉)
- 레이아웃 저장(💾 버튼): 항상 data_[번호].json에만 저장
- save_data.php 방식: 단일 파일 덮어쓰기 (타임스탬프 이력 파일 미생성)

[관리자 모드]
?admin=1 파라미터로 진입 시에만 [설정 ⚙] 버튼 노출.
isAdminMode()는 URLSearchParams.get('admin') === '1' 방식으로 파싱 (버그 수정 완료).
