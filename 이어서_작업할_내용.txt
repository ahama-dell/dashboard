2. 📁 파일 구조 및 NAS 업로드 경로 매핑
────────────────────────────────────────────────────────────────
작업 중인 로컬 컴퓨터의 파일들을 시놀로지 NAS의 아래 폴더에 업로드해야 합니다.

[웹 스테이션 폴더] ➡️ NAS 경로: `/volume1/web/dashboard/`
  * production_dashboard.html  ➡️  index.html (이름 변경하여 업로드)

  * data.json                  ➡️  data.json

[파이썬 백그라운드 프록시 폴더] ➡️ NAS 경로: 임의의 작업 폴더 (예: `/volume1/web/dashboard/`)
  * proxy_server.py  (tempcare.kr에서 실시간 온습도 데이터를 수집하는 프록시 서버)
  * start.sh         (시놀로지 작업 스케줄러에서 프록시 서버를 시작하는 쉘 스크립트)
  * session.txt      (tempcare.kr의 브라우저 세션 키 - JSESSIONID 보관용 파일)

────────────────────────────────────────────────────────────────
3. ⚠️ 해결된 주요 문제점 및 특이사항
────────────────────────────────────────────────────────────────
* [V3 보안 프로그램 충돌 우회]: 
  포트 8090이나 8765 같은 비표준 포트를 통해 1.7MB가 넘는 대용량 파일(data.json)을 저장할 때,
  안랩 V3 등 PC의 보안 프로그램이 이를 차단하여 '서버 저장 실패(net::ERR_CONNECTION_RESET)' 에러가 발생했었습니다.
  이를 우회하기 위해, 저장(POST) 기능은 표준 웹 포트인 8080(내부 80)의 'save_data.php'를 거치게 설계하고,
  실시간 온습도 데이터(GET)만 8090(내부 8765) 포트의 파이썬 API 서버를 찌르도록 이원화했습니다.
  이로 인해 외부망에서도 저장과 실시간 데이터 수집이 모두 완벽하게 작동합니다.

────────────────────────────────────────────────────────────────
4. 🚀 다음에 작업할 내용 (메인 포털 판넬 구축 계획)
────────────────────────────────────────────────────────────────
다음 작업 시에는 사용자가 주소창에 포트(:8080)만 쳤을 때 열리는 
"통합 메인 화면(index.html)"을 제작할 예정입니다.

* 파일 위치: NAS의 `/volume1/web/index.html` (dashboard 폴더 밖의 최상위 web 폴더)
* 주요 기능:
  ① 세련된 다크모드 및 카드(Card) 디자인 적용.
  ② 양평동 생산현황 대시보드(dashboard/index.html)로 바로 이동하는 메뉴/아이콘 제공.
  ③ 실시간 온습도 모니터링(dashboard/temp.html)으로 바로 이동하는 메뉴/아이콘 제공.
  ④ 추후 타 공사 현장 대시보드가 추가될 때 카드를 쉽게 확장하여 붙일 수 있도록 설계.
